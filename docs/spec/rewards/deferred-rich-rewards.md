## 位置づけ

MVP の方針：

- 特典は **コードだけで完結** する 3 種に絞る（SVG バッジ・達成カード PNG・Hall of Fame）。
- 3D・Lottie・AI 生成・自動 SNS 投稿はすべて運用負荷（制作・継続コミット・API コスト・SNS API トークン管理）が高いため Phase 2 以降に保留。
- UI 上では **「Coming Soon」プレースホルダ枠** を用意して、ユーザーに将来追加される予告を見せる。

## 着手トリガー

以下のいずれかが発生したときに本ドキュメントの設計案を検討し、実装計画を立てる：

1. **MAU 1000 名突破** → ユーザーに見せる価値があるリッチ特典を追加するモチベーションが出る
2. **広告収益が月 $50 以上に到達** → AI 生成コスト（月 $20 程度）を負担できる
3. **ユーザーからのリクエスト**：
   - 「3D アバターが欲しい」「動くアイコンが欲しい」
   - 「カッコいい特典カードが欲しい」
   - 「公式アカウントが上位者を紹介してくれたら嬉しい」
4. **コミュニティ・SNS で「特典が地味」という指摘** が一定数発生したとき

これらが起きない限り、MVP の素朴な 3 種で運用を続ける。

## 設計案

### 3D オリジナルキャラの内製・配布

エンジニアキャリア風グレード（Intern → Fellow）に連動した **8 体の 3D キャラ** をフリーランス 3D アーティストに制作依頼。グレードアップでキャラが解放されコレクション要素になる。

- 制作費：1 体あたり 5〜10 万円、8 体で 40〜80 万円の初期費用
- 配布形式：PNG（プロフィール画像用）+ GLB（VRChat / Three.js 表示用）
- ステータス合成：個別の「色相シフト」「ベストスコア表示」で全員違うバリエーション
- 配信：S3 + CDN、ファイルサイズ 1 MB 以下に最適化
- 運用：キャラ追加・微調整時に 3D アーティストに発注し直し

### Lottie アニメーションアバター

静止画ではなく **動くアバター** を提供。Twitter / Discord プロフィールでも動く。

- 制作費：1 グレードあたり 3〜5 万円（モーションデザイナー発注）
- ファイルサイズ：数 KB〜数十 KB（Web 配信に最適）
- 配信形式：Lottie JSON / GIF / MP4 の 3 形態
- 配信：CDN 経由
- 運用：8 グレード × アニメーション 24〜40 万円の初期費用

### AI トレーディングカード

カードゲームのレアカード風 PNG。トップ 10 入りなど **特別な瞬間のみ** に発行する希少特典。

- 背景：AI 生成（DALL-E 3 / Imagen 3）で「TypeScript 神」のような世界観
- フレーム：プロシージャル（グレードでレア度演出、ホロ加工）
- ステータス：bestScore / accuracy / 順位 を文字で配置
- 1 枚あたりコスト：DALL-E 3 standard $0.04 / HD $0.08
- 月コスト試算：トップ 10 入り 月 20 枚 = **月 $0.80** ／ グレードアップ全員 月 500 枚 = **月 $20**
- 運用：OpenAI / Google AI Studio のトークン管理

### プロシージャル識別子アート

`userId + grade + bestScore` を **シード値** にしてコードで決定論的に画像を生成する方式。AI 不要、運用コストゼロ。

- 実装：Three.js / Canvas で生成
- 例：パーティクル、フラクタル、低ポリゴンアバター、glitch art、identicons 強化版
- 同じユーザーは何度生成しても同じ結果 → 自分のユニーク識別子になる
- ファイル生成：サーバー側 Headless Three.js / Skia でレンダリング
- 「ハッカー美学」と相性が良い

### 上位者の X 自動投稿

**プロダクト公式 X アカウント** が、トップ 10 入り / グレードアップ / 累計達成 などの節目で **上位者を紹介する自動投稿** を行う仕組み。

#### 投稿例

```
🏆 TypeScript 部門 #3 達成！
@kenta_fujimori さん（Senior Engineer）
ベストスコア 543 / 正確率 98.2%
今回のチャレンジ：React のソースコード

リプレイを見る: https://...
#コードタイピング #TypeScript
```

#### 起動条件（暫定）

| 条件 | 投稿内容 |
| --- | --- |
| 月間オールタイムトップ 10 初入り | 「{username} さんがトップ 10 入り！」 |
| グレード Staff Engineer 以上に到達 | 「{username} さんが {grade} に昇格！」 |
| 累計 100 万文字達成 | 「{username} さんが 100 万文字到達！」 |
| 神々モードで神を破った | 「{username} さんが神 {godUsername} を打ち破った！」 |

#### 仕様

- ユーザーは **マイページの設定でオプトイン**（既定 OFF）にしないと投稿対象にならない。
- ユーザーが X アカウントを連携している場合は `@{handle}` でメンション、未連携なら表示名のみ。
- 投稿頻度：1 アカウントあたり 1 日 1 回まで（スパム防止）。
- リプレイ URL・達成カード画像を添付。

#### 技術設計

- **X API v2（有料）** を利用：Basic プラン $100/月、Pro プラン $5,000/月
- 公式アカウントの **App-only OAuth 2.0** で投稿
- 投稿キュー：Redis で順次処理、API レート制限（500 tweet/月 Basic）を尊重
- 達成イベント → ジョブキュー投入 → ワーカーが順次投稿
- 投稿履歴を `social_posts` テーブルに記録（再送防止、ユーザーからのリクエストで削除可能）

#### コスト感

- Basic $100/月で 500 tweet 投稿可能
- 月 100 名分の達成投稿で十分賄える試算
- 広告収益が月 $200 以上に達したら導入検討

#### 個人情報・規約面

- X 利用規約に違反しないか確認（自動投稿のレート制限・スパム判定回避）
- ユーザーオプトインを必須に。利用規約とプライバシーポリシーに明記
- アカウント削除時は過去投稿への削除リクエストも自動キュー
